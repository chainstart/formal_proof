# Formal Proof Support Materials

This repository is the public archive for Lean formal proof support packages
referenced by manuscripts.  The repository itself is named `formal_proof`;
each theorem or paper artifact lives in its own top-level subdirectory.

## Repository Layout

Each package is intended to be reproducible on a fresh machine and should
contain:

- the relevant Lean source files and `lakefile.toml`;
- `lean-toolchain` and `lake-manifest.json`, when present;
- `DEPENDENCIES_AND_REPRODUCTION.md` with exact setup and build commands;
- `LOCAL_VERIFICATION.md` or `verification_report.json` recording the local
  verification result;
- `ARTIFACT_MANIFEST.json` describing the package, theorem names, source file,
  public URL, and build status.

Local build outputs are intentionally not tracked.  In particular, `.lake/`
should be regenerated by the reader:

```bash
cd <package>
lake exe cache get
lake build <target-module>
```

## Adding A Package

Add new proof support material as a new top-level directory rather than placing
it directly at the repository root.  Use a short ASCII slug, for example
`wowii_local_girth_induced_tree/`, and update the package table above.  Keep
generated caches, editor files, local agent logs, and private working notes out
of the committed artifact.
